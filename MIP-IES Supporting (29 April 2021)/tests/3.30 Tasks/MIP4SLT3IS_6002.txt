Test description

Test IDMIP4SLT3IS_5903Test TitleCreate Task with status information.Execution Priority1ObjectiveBSO validation. Test whether the Producer/Provider is able to create and update the status information of a Task (AttackByFire for instance) and the Consumers/Receivers are able to understand (and display) it correctly.ScenarioThe Producer creates a Task (AttackByFire) with specific status information.
The Consumers receive the information and interpret it correctly.
The Producer updates the status information of this AttackByFire.
The Consumers receive the information and interpret it correctly.EnvironmentInternet or Co-located. Exchange Pattern: R/R, P/S.Participation2 or more.MTRSApplicablePre-test ConditionsThe Producer/Provider has created a topic which includes message elements belonging to the Task message schema area.
The Consumers have subscribed to this topic.Test InputsN/A.ConclusionThis test is considered a success if the AttackByFire and its status information is processed and interpreted correctly on all Nations' C2IS.Test OutputsN/A.TraceabilityREQ_INF_0002
